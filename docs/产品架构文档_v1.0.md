## 一、项目定位

**产品名称：** KPOP Global Platform（暂定）

**核心定位：**
> AI驱动的全球KPOP实时活动与数据平台

**不是：**
- ❌ 普通韩娱新闻站
- ❌ WordPress韩娱博客
- ❌ 没有数据能力的展示站

**是：**
- ✅ 全球KPOP活动数据库
- ✅ AI自动内容生成系统
- ✅ 多语言实时信息平台
- ✅ 未来可扩展至APP/会员系统/AI助手

## 四、数据库结构（Supabase/PostgreSQL）

### 4.1 艺人表 (artists)

```sql
CREATE TABLE artists (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name_ko VARCHAR(100),        -- 韩文名
  name_zh VARCHAR(100),        -- 中文名
  name_en VARCHAR(100),        -- 英文名
  group_name VARCHAR(100),     -- 团体名（如果是成员）
  agency VARCHAR(100),         -- 经纪公司
  country VARCHAR(50),         -- 所属国
  debut_date DATE,            -- 出道日期
  profile_image TEXT,         -- 头像URL
  banner_image TEXT,          -- 横图URL
  social_links JSONB,         -- SNS链接 {weverse, twitter, instagram, youtube}
  member_count INT,           -- 成员数
  tags TEXT[],               -- 标签 [流行, Kpop, 女团]
  status VARCHAR(20) DEFAULT 'active', -- active, hiatus, disbanded
  hot_score INT DEFAULT 0,   -- 热度指数
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 4.2 活动表 (events)

```sql
CREATE TABLE events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title VARCHAR(200),
  artist_id UUID REFERENCES artists(id),
  event_type VARCHAR(30),      -- concert, fansign, pop-up, fanmeeting, festival, broadcast
  country VARCHAR(50),
  city VARCHAR(100),
  venue VARCHAR(200),
  address TEXT,
  latitude DECIMAL(10, 8),
  longitude DECIMAL(11, 8),
  event_date TIMESTAMPTZ,
  ticket_open_date TIMESTAMPTZ,
  price_range VARCHAR(50),    -- "0-500000"
  currency VARCHAR(10),        -- KRW, CNY, USD
  official_link TEXT,
  poster_image TEXT,
  status VARCHAR(20) DEFAULT 'upcoming', -- upcoming, ticketing, ended, cancelled
  hot_score INT DEFAULT 0,
  source VARCHAR(50),         -- 抓取来源
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_events_date ON events(event_date);
CREATE INDEX idx_events_country ON events(country);
CREATE INDEX idx_events_artist ON events(artist_id);
```

### 4.3 新闻表 (news)

```sql
CREATE TABLE news (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title VARCHAR(300),
  summary TEXT,               -- AI生成摘要
  content TEXT,
  source VARCHAR(100),
  source_url TEXT,
  language VARCHAR(10),        -- ko, zh, en, ja
  original_title TEXT,        -- 原文标题
  original_content TEXT,
  artist_id UUID REFERENCES artists(id),
  news_type VARCHAR(30),      -- comeback, event, achievement, controversy, schedule
  tags TEXT[],
  image_url TEXT,
  publish_date TIMESTAMPTZ,
  is_translated BOOLEAN DEFAULT FALSE,
  is_ai_summary BOOLEAN DEFAULT FALSE,
  status VARCHAR(20) DEFAULT 'pending', -- pending, published, archived
  hot_score INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_news_publish ON news(publish_date DESC);
CREATE INDEX idx_news_artist ON news(artist_id);
```

### 4.4 用户表 (users)

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  nickname VARCHAR(50),
  avatar_url TEXT,
  language VARCHAR(10) DEFAULT 'zh',
  country_filter VARCHAR(50)[], -- 用户关注的国家
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 4.5 收藏表 (favorites)

```sql
CREATE TABLE favorites (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  artist_id UUID REFERENCES artists(id),
  event_id UUID REFERENCES events(id),
  notify_before INT,          -- 提前N小时提醒
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, artist_id),
  UNIQUE(user_id, event_id)
);
```

### 4.6 快速讯号表 (flash_news)

```sql
CREATE TABLE flash_news (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content TEXT,               -- AI生成的快讯内容
  news_type VARCHAR(30),     -- comeback, announcement, ticketing, event
  artist_id UUID REFERENCES artists(id),
  priority INT DEFAULT 5,    -- 1-10, 越高越重要
  is_active BOOLEAN DEFAULT TRUE,
  expires_at TIMESTAMPTZ,    -- 过期时间
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
